## 🔴 우선순위 높음 — 실제 동작 검증

### 1. 브라우저에서 실제 파이프라인 테스트
현재 코드가 빌드는 되지만 **실제로 동작하는지 아직 미확인** 상태입니다.
- CORS 프록시(`allorigins.win`, `corsproxy.io`)가 네이버 RSS/PostView HTML을 실제로 가져오는지
- LLM이 `IMAGE_N` 패턴 프롬프트에서 올바른 `image_indices`를 반환하는지
- `CelebItem` 추출 결과가 `ItemsPanel`에 정상적으로 표시되는지

## 🟡 우선순위 중간 — 기능 보완

### 2. 이미지 CORS 문제 해결
`ItemsPanel`에서 `pstatic.net` 이미지를 `<img>` 태그로 직접 불러오면 **브라우저 CORS 정책에 막혀** 깨질 가능성이 높습니다.
- 이미지도 CORS 프록시를 거쳐서 불러오도록 처리 필요
- 또는 이미지 URL만 표시하고 클릭 시 새 탭으로 여는 방식으로 대체

### 3. `vvd.bz` 단축 URL 해결
구매 링크(`link_url`)가 `vvd.bz` 단축 URL인 경우 브라우저에서 리다이렉트 추적이 불가합니다.
- 현재는 단축 URL 그대로 저장
- 백엔드(`pipeline/collect_fixtures.py`)에서는 해결 가능하므로, 백엔드 활용 방안 고려

### 4. 연예인 필터링 정확도 개선
Step 3에서 `title.includes(celeb)` 단순 문자열 매칭으로 포스트를 필터링하는데, **연예인 이름 변형이나 오탈자**에 취약합니다.
- 퍼지 매칭 또는 LLM 기반 필터링 고려
